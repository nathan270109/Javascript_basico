Estrutura if/else

if/else é a estrutura mais básica de decisão em programação. if verifica uma condição e executa um bloco se for verdadeira. else executa outro bloco se for falsa.

let idade = 16; if (idade >= 18) { console.log("Maior"); } else { console.log("Menor"); }

Múltiplos else if

Podemos encadear várias condições com else if. O programa verifica cada condição em ordem e executa apenas a primeira que for verdadeira.

let nota = 85; if (nota >= 90) { console.log("A"); } else if (nota >= 80) { console.log("B"); } else { console.log("C"); }

Switch case

Switch é ideal para comparar uma variável contra vários valores específicos. case define cada valor possível, break interrompe a execução, default é o caso padrão.

let dia = 3; switch(dia) { case 1: console.log("Segunda"); break; case 2: console.log("Terça"); break; default: console.log("Outro"); }

Operador Ternário para decisões simples

O ternário é uma forma compacta de if/else em uma única linha: condicao ? valorSeVerdadeiro : valorSeFalso. Use para decisões simples, não para lógicas complexas.

let resultado = idade >= 18 ? "Adulto" : "Menor";

Aninhamento de estruturas

Podemos colocar um if dentro de outro if. Isso é chamado de aninhamento. Cuidado: muitos níveis deixam o código difícil de ler.

if (logado) { if (admin) { console.log("Painel admin"); } }

Condições com lógicos combinados

Use &&, || e ! para criar condições mais poderosas. && exige que todas sejam verdadeiras. || aceita qualquer uma. ! inverte o resultado.

if (idade >= 18 && temIngresso) { console.log("Pode entrar"); }

Truthy e falsy

Em JavaScript, valores podem ser verdadeiros ou falsos em contexto booleano. São falsy: false, 0, "" (string vazia), null, undefined, NaN. Todo o resto é truthy.

if ("texto") { console.log("É truthy!"); } if (0) { console.log("Não executa"); }

Boas práticas em decisões

Mantenha if/else simples e legíveis. Evite aninhamentos profundos (max 2 níveis). Prefira switch para múltiplos casos fixos. Use ternário apenas para decisões simples de uma linha. Sempre use === em vez de ==. Teste todas as condições possíveis.
