O que é JavaScript?

JavaScript (JS) é a linguagem mais popular da web. Ela roda direto no navegador, sem precisar instalar nada. Com JS podemos criar sites interativos, jogos, animações e validar formulários. É uma linguagem de alto nível (fácil de ler) e interpretada (roda linha por linha).

Ferramentas: navegador e editor

Para programar em JS você só precisa de duas ferramentas: um navegador (Chrome, Firefox, Edge) e um editor de código (VS Code, Sublime, ou até o Bloco de Notas). O navegador executa o código, o editor ajuda a escrever.

Dica: Use o Chrome e aperte F12 para abrir o console.

Seu primeiro programa

Abra o console (F12, aba Console). Digite:

console.log("Olá, mundo!");

e aperte Enter. console.log() exibe mensagens na tela. É como o "print" da programação — a forma mais básica de ver o resultado do seu código.

Sintaxe básica

Sintaxe são as regras de escrita do código. Em JS:

  • Ponto e vírgula no final de cada linha ;
  • Parênteses para funções ()
  • Aspas para textos "" ou ''
  • Chaves para blocos {}

O computador é exigente — um erro de sintaxe e o programa não roda.

Console.log como depurador

console.log() não serve só para exibir mensagens. É a principal ferramenta de depuração (debugging). Você pode exibir valores de variáveis, verificar se uma parte do código executou, e entender o fluxo do programa.

console.log("Chegou aqui!"); console.log("Valor de x:", x);

Pensamento Computacional

É a habilidade de resolver problemas de forma lógica, organizando o raciocínio em passos claros. Os 4 pilares:

  • Decomposição: quebrar em partes
  • Reconhecimento de Padrões: identificar repetições
  • Abstração: focar no essencial
  • Algoritmos: sequência de passos

Comentários no código

Comentários são ignorados pelo computador. Servem para humanos se entenderem.

// comentário de uma linha /* comentário de várias linhas */

Use comentários para explicar a lógica, documentar seu código e organizar seções. É uma boa prática desde o início.

Boas práticas iniciais

Use nomes claros para variáveis, mantenha indentação organizada, comente o código, teste pequenas partes de cada vez. O programador iniciante que cria bons hábitos desde o começo se torna um profissional melhor.
