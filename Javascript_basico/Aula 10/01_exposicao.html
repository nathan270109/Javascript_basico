Planejamento do Projeto

Todo projeto começa com um plano. Defina o objetivo: o que sua aplicação vai fazer? Quem vai usar? Quais são as funcionalidades principais? Desenhe um esboço no papel. Divida em etapas menores. Exemplo: "Quero um app de lista de tarefas com adicionar, remover e filtrar itens."

Estrutura HTML semântica

Use tags HTML com significado: header, main, section, article, footer. Cada parte da página tem uma função. Cabeçalho com título, área principal com conteúdo, rodapé com informações. Exemplo:

<header><h1>Minha Aplicação</h1></header>
<main><section id="lista"></section></main>

Estilização responsiva

Use CSS flexível que se adapta a qualquer tela. Unidades relativas (rem, %, vw, vh). Media queries para ajustar o layout em celulares. Design mobile-first: comece pelo celular e depois expanda. Exemplo:

@media (max-width: 600px) {
.grid { grid-template-columns: 1fr; }
}

Lógica JavaScript central

Organize o JS em funções claras. Variáveis globais para estado, funções para cada ação. Separação: manipulação do DOM separada da lógica de dados. Exemplo:

function adicionarItem(texto) {
itens.push(texto);
renderizarLista();
}

Manipulação do DOM avançada

Crie elementos dinamicamente com createElement. Use event delegation para elementos que ainda vão existir. Atualize apenas partes da página, não recarregue tudo. Exemplo:

lista.addEventListener('click', function(e) {
if (e.target.classList.contains('remover')) {
/* remover item */
}
});

Uso de localStorage

localStorage salva dados no navegador do usuário. JSON.stringify para converter objeto em texto. JSON.parse para recuperar. Os dados persistem mesmo após fechar a página. Exemplo:

localStorage.setItem('meusDados', JSON.stringify(dados));
let dados = JSON.parse(localStorage.getItem('meusDados'));

Testes e depuração

Teste cada funcionalidade isoladamente. Use console.log para verificar valores. O DevTools do navegador (F12) é seu melhor amigo: Console para logs, Elements para HTML, Network para requisições. Teste em vários navegadores.

Publicação no GitHub Pages

GitHub Pages hospeda sites estáticos gratuitamente. Crie um repositório, faça upload dos arquivos, ative o Pages nas configurações. Seu site fica online em https://[usuario].github.io/[repositorio]/. Compartilhe com o mundo!
